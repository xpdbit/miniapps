## 一、游戏模式概述

### 1.1 产品定位

游戏模式是 AI-Tavern 的 **TRPG 风格叙事引擎**。用户以**自己扮演的角色**身份，置身于 AI 角色群中——你是故事中真实存在的人物，而非外部操控者。AI 角色之间自主互动，推动剧情发展；在关键时刻，AI 将镜头转向你，等待你的选择。

**一句话定义**：AI 做 GM 和 NPC，你做 PC。故事自己长出来，你在关键处决定方向。

### 1.2 核心循环

```
┌─────────────────────────────────────┐
│          🎭 AI 自主互动段             │
│  AI 角色自然对话，推动剧情             │
│  • 戏剧冲突 / 世界探索 / 意外趣味      │
│  • AI Script 事件穿插其中             │
└──────────────┬──────────────────────┘
               │ 自然停顿
               ▼
┌─────────────────────────────────────┐
│          📋 AI 总结 + 选项             │
│  简述当前局面，生成 2-3 个可选行动      │
│  [选项A] [选项B] [选项C]               │
│  或 [自定义行动...]                    │
└──────────────┬──────────────────────┘
               │ 用户选择/输入
               ▼
┌─────────────────────────────────────┐
│          🎭 AI 反应段                 │
│  角色回应用户的选择，故事推进           │
│  事件链继续...                        │
└──────────────┬──────────────────────┘
               │
               ▼
           （循环）
```

### 1.3 与酒馆模式的关系

```
酒馆模式                          游戏模式
┌─────────────┐                  ┌──────────────────┐
│ 你 ↔ 一个角色 │                │ 你是世界中的一个角色  │
│ 1v1 闲聊    │                │ AI 角色群自主互动    │
│ 你主导话题   │                │ AI 主导叙事，你抉择   │
│ 无世界观     │                │ 完整世界观 + 存档    │
│ 随时开始/结束 │                │ 持续性叙事体验       │
└─────────────┘                  └──────────────────┘
         │                                │
         └────────────┬───────────────────┘
              同一角色卡池 / 同一 AI Provider / 同一认证
```

### 1.4 核心概念速览

| 概念 | 定义 | 类比 |
|------|------|------|
| **存档 (Save)** | 一个完整的游戏世界实例 | TRPG 的一局战役 |
| **世界观 (World Setting)** | 标题 + 描述 + 规则 | 战役设定集 |
| **你的角色 (Persona)** | 用户在游戏世界的化身 | TRPG 中你扮演的 PC |
| **角色卡组 (Selected Cards)** | 选入世界的 AI 角色卡片 | NPC 阵容 |
| **群组 (Group)** | AI 角色（+你）组成的聊天群 | TRPG 的 party |
| **选项 (Choices)** | AI 在停顿点给你的行动建议 | 交互小说的分支选项 |
| **AI Script** | AI 驱动的世界状态变化 | GM 的规则判定 |
| **Scenario** | 预设的完整剧本 | TRPG 的模组 |

## 二、完整用户旅程

### 2.1 端到端流程

```
酒馆模式首页
  │
  ▼
[1] 进入"开始"页面 (game-setup)
  │
  ├── 看到已有存档列表
  │     ├── 继续游戏 → [6]
  │     └── 新建存档
  │
  ▼
[2] 创建新存档 — 命名、设定玩家人数
  │
  ▼
[3] 构建世界观 — 手动填写 / AI 辅助生成
  │
  ▼
[4] 选择角色卡 — CHARACTER / MECHANISM / MAP / BACKGROUND
  │
  ▼
[5] 编排群组 — 分配 AI 角色到群组
  │   确认你的角色化身（从已有人设中选择或新建）
  │
  ▼
[6] 进入游戏模式 — TabBar 切换
  │
  ▼
[7] 群组中开始故事
  │
  ├── AI 角色自主互动（你在看）
  │     └── 角色对话 / 事件触发 / 状态变化
  │
  ├── AI 停顿，总结局面，给出选项
  │     ├── 点击选项 → 你的角色执行行动
  │     └── 或自由输入 → 自定义行动
  │
  └── AI 角色反应 → 故事推进 → 下一段
  │
  ▼
[8] 持续游玩 / 暂停 / 删除存档
```

### 2.2 AI 自主互动段的设计

AI 自主段是用户"观看"的部分。设计目标是让观看本身有趣：

| 维度 | AI 应做到的 |
|------|-----------|
| **戏剧冲突** | 角色之间有不同意见、隐藏动机、情感张力 |
| **世界探索** | 对话中自然揭示世界观、历史、秘密 |
| **意外趣味** | 角色间的反差萌、AI 的创造性发挥、有趣的误会 |
| **信息密度** | 2-4 轮对话内包含有意义的情节推进 |
| **自然收尾** | 以一个问题、困境或抉择结束，自然引向选项 |

### 2.3 存档生命周期

```
         ┌─────────┐
         │  创建   │
         └────┬────┘
              │
         ┌────▼────┐
         │  编辑中  │  世界观、选卡、编组、选择化身
         └────┬────┘
              │
    ┌─────────┼─────────┐
    │         │         │
┌───▼──┐ ┌───▼──┐ ┌───▼──┐
│游玩中 │ │ 暂停  │ │ 删除  │
└───┬──┘ └───┬──┘ └──────┘
    │        │
    │   可重新编辑或继续
    │
退出游戏模式（存档保留）
```

## 三、存档系统设计

### 3.1 GameSave 数据结构

```typescript
interface GameSave {
  id: string
  name: string
  createdAt: number
  updatedAt: number
  playerCount: number
  schemaVersion: number

  userPersonaId?: string        // 用户在游戏中的角色化身 ID

  selectedCards: {
    characters: string[]        // AI 角色
    mechanics: string[]         // 规则卡
    maps: string[]              // 场景卡
    backgrounds: string[]       // 背景卡
  }

  worldSetting: {
    title: string
    description: string
    rules: string[]
  }

  groups: GameGroup[]           // 群组（含用户角色）
}
```

### 3.2 CRUD 操作

| 操作 | 方法 | 持久化 |
|------|------|--------|
| 创建 | `gameStore.createSave(data)` | 立即写 `tavern_saves` |
| 读取列表 | `gameStore.restoreSaves()` | 启动时恢复 |
| 更新 | 各 update 方法 | 即时同步 |
| 删除 | `gameStore.deleteSave(id)` | 移除关联数据 |

### 3.3 存档版本兼容

通过 `schemaVersion` 管理兼容性。恢复时自动迁移旧格式（渐进式、向后兼容、就地升级）。

## 四、世界观构建

### 4.1 两种构建模式

**手动编辑**：结构化表单填写标题、描述、规则。适合有清晰构想的用户。

**AI 辅助生成**：用户输入关键要素 → `aiWorldBuilder` 调用 AI 生成完整世界观 JSON → 审核修改。最多重试 3 次。

### 4.2 设计原则

| 原则 | 说明 |
|------|------|
| 简洁优先 | 3-5 条规则足够，过多让 AI 混乱 |
| 冲突即趣味 | "魔法与科技共存"比"纯魔法世界"有张力 |
| 留白给 AI | 不写死所有细节，给 AI 发挥空间 |
| 用户可控 | AI 生成内容始终可编辑 |

### 4.3 世界观对 Prompt 的影响

世界观通过 Prompt Builder 注入 System Prompt，位于全局提示和角色设定之间。在整个游戏过程中保持不变。

## 五、角色选卡与群组编排

### 5.1 四种卡片在游戏模式中的角色

| 类型 | 角色 | 数量建议 |
|------|------|----------|
| **CHARACTER** | AI 演员，参与群组对话 | 2-6 个 |
| **MECHANISM** | 世界规则，约束 AI 行为 | 0-2 个 |
| **MAP** | 空间语境，丰富场景 | 0-3 个 |
| **BACKGROUND** | 世界观深度，时代基调 | 0-2 个 |

### 5.2 用户角色化身（Persona）

在游戏模式中，用户不是无形的存在——你是世界中的一个**真实角色**。创建存档时需要选择或新建你的角色化身：

```
选择你的化身：
┌──────────────────────────────────────────┐
│ 使用已有 Persona：                         │
│ ○ 流浪剑客 — 沉默寡言，剑术精湛...           │
│ ○ 精灵学者 — 博学多闻，好奇心旺盛...          │
│ ● 商队护卫 — 忠诚可靠，嘴硬心软...            │
│                                          │
│ [+ 新建 Persona]                          │
└──────────────────────────────────────────┘
```

**Persona 在游戏中的作用**：
- 你的角色设定注入 System Prompt（与 AI 角色并列）
- AI 角色认识你——会根据你的设定来互动（"剑客先生，你意下如何？"）
- 选项的生成会考虑你的角色能力（剑客看到战斗选项，学者看到调查选项）

### 5.3 群组编排

```
存档 "中土冒险"
  你的化身：商队护卫 贝尔林
  ├── 群组 "护戒小队"
  │     ├── 👤 你 (贝尔林)
  │     ├── 弗罗多 (AI)
  │     ├── 甘道夫 (AI)
  │     └── 阿拉贡 (AI)
  │
  └── 群组 "洛汗会议"
        ├── 👤 你 (贝尔林)
        ├── 希优顿王 (AI)
        └── 伊欧玟 (AI)
```

**群组自动包含用户角色**。不需要手动把自己加进去——你在创建群组的瞬间就是其中一员。

### 5.4 Prompt 注入结构

```
System Prompt (群组 "护戒小队")：

[世界观] 中土大陆·第三纪元：黑暗笼罩...
[规则] • 魔戒腐蚀携带者 • 精灵力量衰退

[你的化身] 贝尔林：
  人格：商队护卫，忠诚可靠，嘴硬心软
  能力：擅长剑术和野外生存，对中土地理熟悉
  说话风格：直率但不多话，习惯用行动代替言语

[AI 角色] 弗罗多：...
[AI 角色] 甘道夫：...
[AI 角色] 阿拉贡：...

[当前局面] 队伍刚刚逃出摩瑞亚矿坑，正在向罗斯洛立安前进
```

### 5.5 游戏模式三页面

| 页面 | 功能 |
|------|------|
| **通信** | 群组对话中心。列表 + 置顶 + 未读 + 最后消息预览 |
| **通讯录** | 角色关系总览。AI 角色实时状态（位置/HP/关系值）+ 你的化身信息 |
| **发现** | 世界动态。事件时间线 + 任务日志 + 世界状态摘要（时间/天气/维度） |

## 六、群组对话机制（精炼后）

### 6.1 AI 自主互动段

AI 角色之间自然对话，用户观看。每段 2-4 轮对话：

```
[甘道夫]：摩瑞亚的阴影比我想象的更深。我们必须尽快赶到罗斯洛立安。
[阿拉贡]：同意。但队伍需要休息。弗罗多的状态令人担忧。
[弗罗多]：我...我能继续走。只是肩膀上的伤...
[莱戈拉斯]：前方有一片空地，适合扎营。我可以放哨。
```

AI Script 事件在对话中穿插触发（如 `character.move` 更新位置、`stat.modify` 更新弗罗多体力）。

### 6.2 AI 停顿与选项生成

一段自主互动结束后，AI 总结局面并生成 2-3 个行动选项：

```
📋 当前局面
队伍在摩瑞亚出口附近的林间空地。弗罗多受伤疲惫，
阿拉贡主张休息，甘道夫担心追兵。

你的行动：
┌────────────────────────────────────────┐
│ 🏕️ 支持扎营休息                         │
│   帮忙搭建营地，照顾弗罗多的伤势           │
├────────────────────────────────────────┤
│ 🔍 在周围侦查一圈                        │
│   利用你的野外生存经验，确保没有兽人追踪    │
├────────────────────────────────────────┤
│ 🗣️ 与甘道夫讨论下一步路线                 │
│   你对中土地理熟悉，或许有更好的路线建议    │
└────────────────────────────────────────┘
── 或 ──
[ 输入你想做的任何事...________________]
```

**选项生成原则**：
- 每个选项代表不同方向（战斗/探索/社交/撤退），不是同一方向的程度差异
- 选项融入你的角色设定（剑客看到战斗相关，学者看到调查相关）
- 选项有叙事后果——选 A 和选 B 会导致不同的故事发展
- 第三个位置偶尔是"高风险高回报"选项（增加张力）

### 6.3 用户选择后的 AI 反应

```
用户选择：🔍 在周围侦查一圈

→ AI 反应：

[阿拉贡]：我和你一起去。这片林地我来过，知道几个有利的观察点。
（AI Script: character.move → 阿拉贡和你移动到林地边缘）

[阿拉贡]：（低声）看那边——狼骑兵的踪迹。不超过两小时的路程。
（AI Script: world.trigger_event → enemy_nearby）

[阿拉贡]：我们得赶快回去通知甘道夫。今晚不能生火了。

→ 回到营地后，AI 自然进入下一段互动，最终再次停顿给出选项。
```

### 6.4 自由输入兜底

选项下方始终有自由输入框。用户可以：
- 忽略选项，输入完全自定义的行动（"我悄悄跟上阿拉贡，但保持距离"）
- 在 AI 自主段中随时插入（不等选项出现）

自由输入的消息作为用户角色的发言/行动，AI 角色自然回应。如果是在选项出现后输入，AI 将忽略未选择的选项，直接回应用户输入。

### 6.5 对话上下文管理

```
上下文窗口：
┌─────────────────────────────────┐
│ System Prompt（固定）             │ 世界观 + 你的化身 + AI角色 + 规则
├─────────────────────────────────┤
│ 历史消息（滑动窗口）               │ 最近 N 轮对话
├─────────────────────────────────┤
│ AI Script 状态（动态）            │ 位置/属性/时间/天气
├─────────────────────────────────┤
│ 选项上下文                       │ 当前局面总结 + 2-3 个选项文本
├─────────────────────────────────┤
│ 用户选择 / 自由输入               │
└─────────────────────────────────┘
```

### 6.6 选项生成的 AI Prompt 设计

在 AI 回复完成一轮自主互动后，追加选项生成指令：

```
你现在需要暂停叙事，将镜头转向玩家的角色"贝尔林"。

请：
1. 用一句话总结当前局面（第三人称）
2. 生成 3 个玩家可选的不同方向的行动，每个包含：
   - 一个简短的行动名称（5-10 字）
   - 一个一句话描述（15-30 字）
3. 这些选项应：
   - 代表明显不同的方向（不要都是"同意"的变体）
   - 考虑贝尔林的角色设定（商队护卫，擅长剑术和野外生存）
   - 第三个选项偶尔提供高风险/意外方向
   - 每个选项都应有有意义的叙事后果

以 JSON 格式返回：
{
  "summary": "队伍在林地间空地...",
  "choices": [
    { "label": "支持扎营", "description": "帮忙搭建营地，照顾弗罗多" },
    ...
  ]
}
```

### 6.7 异常处理与降级

| 异常 | 处理 |
|------|------|
| AI 超时 60s | 自动重试 1 次 → 提示用户切换模型 |
| Provider 不可用 | 自动切换到免费模型 |
| 选项 JSON 解析失败 | 降级为纯文本对话（无选项），用户自由输入 |
| 网络断开 | 消息入本地队列，恢复后自动发送 |
| 配额耗尽 | 阻止发送，提示升级或等待明天 |

### 6.8 消息编辑与对话分支

用户编辑已发送的消息后，该消息之后的所有 AI 回复和选项将被清除，AI 基于新输入重新生成。这是"对话分支"——你可以回溯到某个节点，做出不同选择。

## 七、AI Script 事件系统

### 7.1 事件驱动理念

AI 对话不只是文本——角色可以移动、属性可以变化、世界可以推进。AI Script 让 AI 在对话中触发游戏世界事件。

```
纯文本：用户 ↔ AI（无状态）
AI Script：用户 ↔ AI → 事件 → 状态更新 → 影响后续
```

### 7.2 14 种事件类型

| 事件 | 效果 | 示例 |
|------|------|------|
| `character.move` | 角色移动到新位置 | 阿拉贡移动到林地边缘 |
| `character.action` | 角色执行动作 | 甘道夫施放照明术 |
| `stat.modify` | 属性变化 | 弗罗多 HP-5 |
| `item.add/remove` | 物品增减 | 获得精灵宝剑 |
| `world.advance_time` | 时间推进 | +8h，进入夜晚 |
| `world.change_weather` | 天气变化 | 起浓雾 |
| `world.trigger_event` | 全局事件标记 | 龙族苏醒 |
| `dimension.modify` | 维度切换 | 传送至暗影界 |
| `relation.change` | 关系变化 | 阿拉贡对你的信任+5 |
| `quest.start/progress/complete` | 任务管理 | 主线推进 2/5 |

### 7.3 GameStateStore

服务端内存状态管理器。记录所有角色的位置、属性、物品、关系，以及世界时间、天气、任务进度。通过事件溯源重建状态——所有事件按序记录到日志，重启时重放。

### 7.4 事件链：一场伏击

AI 自主互动中，事件自然串联成叙事弧：

```
AI 段开始 → 甘道夫察觉危险 (character.action)
         → 兽人包围 (character.move orcs)
         → 天气变为战斗烟尘 (world.change_weather)
         → [停顿，给你选项：正面迎战 / 绕路 / 让甘道夫用魔法]
         → 你选择正面迎战
         → 阿拉贡杀敌×5，HP-10 (stat.modify)
         → 甘道夫 MP-15 (stat.modify)
         → 战斗结束，伏击生还 (world.trigger_event)
         → 任务进度+1 (quest.progress)
         → 时间+3h，夜晚 (world.advance_time)
         → [暂停，给下一组选项]
```

### 7.5 事件与选项的关系

AI 在生成选项时，会考虑当前 GameWorldState：
- 如果阿拉贡 HP 低，选项中可能出现"建议阿拉贡退后休息"
- 如果有未完成的任务，选项中可能出现与任务目标相关的行动
- 如果天气恶劣，选项中可能出现寻找避风处的行动

## 八、Scenario 弹性剧本

### 8.1 剧本结构

预定义的 JSON 剧本包含：meta（元信息）、world（世界观）、characters（角色定义）、events（预设事件）、rules（自定义规则）、prompt（风格指南）。

### 8.2 加载流程

`ScenarioLoader.load()` → Zod 校验 → 解析 cardRef 引用 → 合并角色卡 → 初始化 GameWorldState → 构建 Prompt → 返回 GameContext。用户可在自动填充后手动修改。

### 8.3 与 AI Script 的关系

Scenario 是静态乐谱（"这个世界有什么"），AI Script 是动态指挥（"正在发生什么"），GameWorldState 是演奏中的音乐。

### 8.4 自定义剧本创作

支持表单创作器（推荐）和 JSON 编辑器（高级）两种方式。包含结构校验、角色引用检查、事件逻辑验证、规则冲突检测。支持 V2 JSON 格式导入导出，含版本号管理。

## 十、游戏平衡与 AI 行为约束

### 10.1 AI 行为围栏（六层）

1. **全局角色扮演规则** — "你是一个角色扮演 AI..."
2. **世界观规则** — "在这个世界中，魔法消耗法力值"
3. **角色人格约束** — 每个角色的设定和说话风格
4. **群组动态约束** — 角色轮流发言，不垄断对话
5. **AI Script 事件守卫** — HP<=0 不能战斗
6. **Scenario dos/donts** — 剧本特定的行为准则

### 10.2 选项质量保障

| 问题 | 对策 |
|------|------|
| 选项太相似 | Prompt 要求"明显不同的方向" |
| 选项不对角色 | 注入用户 Persona 到选项生成 Prompt |
| 选项无后果 | Prompt 要求"每个选项有叙事后果" |
| AI 带偏向 | 不标注"推荐"选项，三个选项平等呈现 |

### 10.3 内容安全

敏感词过滤 + 用户举报 + NSFW 标记 + Scenario 审核。战斗描述在"有张力"和"不过度"之间平衡。

### 10.4 离线与性能

存档数据在客户端 localStorage，查看历史支持离线。发送消息和 AI 交互需在线。消息超 500 条分页加载；群组角色不超 6 个防 token 超限。
