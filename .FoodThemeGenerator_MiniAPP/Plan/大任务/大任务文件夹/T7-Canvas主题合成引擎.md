# T7: Canvas主题合成引擎

- **波次**: 2
- **预估时长**: 4-5 小时
- **依赖**: T1, T3, T6
- **阻塞**: T14, T16, T17

## 做什么

1. 在客户端实现基于 Canvas 2D 的图片合成引擎（`src/utils/canvas/composer.ts`）
2. 核心合成流程:
   - 加载原始食物图片到 Canvas
   - 加载主题框模板图片（PNG透明背景的UI框，如星露谷对话框架）
   - 将食物图片缩放/裁剪适配到模板的预留区域
   - 叠加模板图层（UI框、文字边框、装饰元素）
   - 生成文字描述叠加到指定位置
   - 导出为最终图片（PNG格式，1080x1080px）
3. 实现主题模板加载器（`src/utils/canvas/templateLoader.ts`）: 支持从云存储加载模板 + JSON配置 + 缓存策略
4. 创建至少3个默认主题模板:
   - **星露谷物语风格**: 对话框架 + 像素字体 + 棕色木质边框
   - **饥荒风格**: 手绘风格边框 + 哥特字体 + 暗色滤镜
   - **塞尔达烹饪风格**: 金色边框 + 古代文字风格 + 火焰装饰
5. 实现文字渲染: 支持自定义字体、颜色、描边、阴影
6. 实现滤镜效果（可选）: 棕褐色调、暗角、像素化、色相调整
7. 创建主题模板配置格式标准（JSON Schema）

## 绝对不能做

- ❌ 不要在主线程中进行大量Canvas操作
- ❌ 不要硬编码模板参数（从JSON文件加载）
- ❌ 不要在合成图片中留白边或错位

## 推荐Agent配置

- **类别**: `visual-engineering` — Canvas合成是纯视觉工程
- **技能**: `[]`

## 验收标准

- [ ] 3个默认主题模板均可正常合成并导出图片
- [ ] 合成图片尺寸为1080x1080px
- [ ] 食物图片在模板中位置正确
- [ ] 文字渲染清晰可读
- [ ] 模板JSON配置文件格式符合Schema定义
- [ ] 模板可从云存储动态加载
