# T14: 主要UI页面（首页+拍照页+结果页）

- **波次**: 3
- **预估时长**: 4-5 小时
- **依赖**: T7, T9, T10, T11, T12
- **阻塞**: T18

## 做什么

1. 创建首页（`src/pages/home/`）:
   - 顶部横幅: 今日打卡状态
   - 主操作按钮: "拍照记录"大按钮（醒目CTA）
   - 最近记录流: 展示最新3条食物记录卡片
   - "查看全部"入口 → 跳转历史页面
   - 底部导航栏: 首页、历史、统计、我的
2. 优化拍照页面（整合T9相机系统）:
   - 整合拍照流程: 拍照 → 预览 → 确认 → 上传 → 跳转结果页
   - 添加加载状态: 上传进度; AI处理步骤进度条
3. 创建结果展示页面（`src/pages/result/`）:
   - 展示合成的主题图片（大图，居中）
   - 展示AI识别结果: 食物名称、热量、类型
   - 展示AI游戏描述
   - 操作按钮组: "保存记录"、"重新拍摄"、"分享"、"查看历史"
   - 主题切换按钮
4. 实现页面间流畅转场动画
5. 实现全局Loading组件和Toast提示组件

## 绝对不能做

- ❌ 不要在首页展示过多无关信息
- ❌ 不要在结果页卡顿（使用懒加载）
- ❌ 不要在AI处理中让用户无法操作

## 推荐Agent配置

- **类别**: `visual-engineering` — 首页和结果页是用户第一印象
- **技能**: `[]`

## 验收标准

- [ ] 首页正常渲染，包含打卡状态、拍照按钮、最近记录
- [ ] 拍照→AI处理→结果展示 完整流程可走通
- [ ] 结果页正确显示主题图片和AI结果
- [ ] 结果页所有操作按钮可点击并响应
- [ ] 底部导航栏切换正常
- [ ] AI处理中显示进度条
