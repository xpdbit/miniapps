# T13: 位置与打卡系统

- **波次**: 3
- **预估时长**: 3-4 小时
- **依赖**: T1, T2, T6
- **阻塞**: T15

## 做什么

1. 实现IP定位功能（云函数 `getLocationByIP`）: 调用第三方IP定位API，解析省份、城市、区县
2. 实现GPS精确定位: 使用 `Taro.getLocation({ type: 'gcj02' })`，优先GPS，失败时降级IP
3. 实现位置名称解析（逆地理编码）: 使用腾讯位置服务将坐标转为地址
4. 创建打卡页面（`src/pages/checkin/`）: 地图组件 + 地址文本 + "在此打卡"按钮
5. 实现打卡记录: 自动打卡（创建食物记录时同步）+ 手动打卡
6. 实现打卡统计: 连续打卡天数、打卡总数、地点分布
7. 创建地点列表: 用户打卡过的所有地点，按打卡次数排序

## 绝对不能做

- ❌ 不要在没有用户授权的情况下获取GPS位置
- ❌ 不要在IP定位失败时阻塞食物记录创建
- ❌ 不要在前端存储完整的位置历史

## 推荐Agent配置

- **类别**: `quick` — 位置服务有成熟的第三方API
- **技能**: `[]`

## 验收标准

- [ ] IP定位返回正确的省份和城市
- [ ] GPS定位精度在合理范围
- [ ] 地图组件正常显示当前位置
- [ ] 打卡功能正常，记录写入 checkins 集合
- [ ] 连续打卡天数计算正确
- [ ] 地点列表按打卡次数正确排序
